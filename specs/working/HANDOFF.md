## 7 Convergence Points (All 3 Agents Agreed)

### 1. Content Dedup Is Broken
- Current: `content_id = SHA-256(input + output)` — same prompt with different responses = no dedup
- Also has concatenation collision bug: `"ab"+"cd"` hashes same as `"abc"+"d"`
- **Proposed fix:** Separate `input_content_id` and `output_content_id` foreign keys

### 2. No shutdown() / Flush API
- Daemon thread dies with process, losing queued spans
- **Proposed fix:** `agentc.shutdown(timeout_ms)` + `atexit` handler + flush on root `@trace` span end

### 3. Use wrapt for Monkey-Patching
- **User approved.** Replace raw setattr with `wrapt.wrap_function_wrapper`. Gets `uninstrument()` for free.

### 4. SDK Version Compatibility Unspecified
- Patch targets are internal module paths that break across SDK versions
- **Proposed fix:** Version-dispatched adapter layer + httpx transport-level fallback

### 5. SimHash → MinHash (but bigger question: hashing at all?)
- MinHash strictly better than SimHash (F1=0.95 vs 0.85, actually faster)
- But hashing misses semantic duplicates and is thrown off by metadata changes
- **Proposed fix:** Skip hashing at capture time in V1. Add MinHash + optional embeddings (model2vec/semhash) at analysis time in V1.1.

### 6. `agentc record` Mechanism Undefined
- Primary zero-code-change CLI entry point has no implementation description
- **Proposed fix:** `sitecustomize.py` approach — create temp dir with sitecustomize that calls `agentc.init()`, prepend to PYTHONPATH, exec user command. Works with pytest/gunicorn/uvicorn.

### 7. Multi-Process SQLite Writes Will Contend
- WAL mode serializes writers, no busy_timeout configured
- **Proposed fix:** Per-process SQLite files (`~/.agentc/active/pid-XXXXX.db`) with merge-on-read. Zero contention. Merge into canonical `traces.db` on process exit via `shutdown()`.

### Rust vs Python Boundary (Decision #5 — Detail)

User asked "shouldn't we go full Rust?" The recommended split:

| Component | Language | Rationale |
|---|---|---|
| SDK patches, decorators, context propagation | **Python** | Must be Python — wrapping Python SDK objects |
| Span creation, queue management | **Python** | Thin layer, feeds Rust core |
| Storage (SQLite writes, compression, content dedup) | **Rust (PyO3)** | Build once, avoid rewrite from Python later |
| Analysis, waste detection, OTLP export | **Rust** | Where performance matters |
| CLI | **Rust** | Single binary, no Python runtime needed |

Python layer should be as thin as possible. Install via maturin pre-built wheels (Linux x86_64/aarch64, macOS x86_64/arm64, Windows x86_64).

2. **Content dedup by SHA-256(input+output) is flawed** — see above.
3. **SQLite multi-process writes will cause data loss** — see above.

### Recommended Changes (from Architecture Review agent)
1. Use `wrapt` (**approved**)
2. Add `agentc.shutdown()` / flush API
3. SDK version compatibility contract + detection
4. Defer SimHash/MinHash to analysis time (not capture time)
5. Fix content_id concatenation collision
6. Handle OpenAI `stream_options` injection carefully (modifies user's API call)
7. Background writer needs backpressure (bounded queue + drop policy)

2. Defer cross-process context propagation to V1.1
3. Defer SimHash/MinHash to V1.1 analysis time
4. Drop `traces` table (derive from spans via view)
5. Simplify buffer to single bounded queue with drop policy

### Things the Spec Gets RIGHT (don't change)
1. Hybrid instrumentation (auto + explicit)
2. OTel gen_ai.* semantic conventions alignment
3. Fail-open error handling
4. Content-addressed storage with compression
